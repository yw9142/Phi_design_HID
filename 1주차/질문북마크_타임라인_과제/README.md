# 질문 북마크 타임라인 (Question Bookmark Timeline)

HID 과제 산출물 — LLM 채팅 UX의 인지적 마찰을 해소하는 새로운 입출력 구조.
아이디에이션 1순위 후보(적합도 5 · 프로토타입 가능성 · 확장성)를 선정해 진행했다.

## 한 줄 정의
실시간으로 흐르는 강의/회의를 들으며 LLM을 쓸 때, **흐름을 끊지 않고 ‘막힌 순간’을 한 번 탭**하면
시스템이 **직전 20~30초 맥락을 자동 저장**하고, 질문은 나중에 그 **시간 맥락 카드** 위에서 고르거나 다듬는다.

## 핵심 마찰
채팅 UX는 질문이 발생한 **‘시간 맥락’을 보존하지 못한다.** 그래서 사용자는
(a) 지금 질문하느라 흐름을 놓치거나, (b) 나중에 방금 맥락을 기억해 다시 설명해야 한다.
→ 음성 입력으로 바꿔도 해결되지 않는, **인터랙션 구조의 문제.**

## 파일
| 파일 | 내용 |
|------|------|
| `prototype.html` | 작동하는 인터랙티브 프로토타입 (단일 HTML, 빌드 불필요) |
| `1Pager_질문북마크타임라인.docx` | 문제 정의 + 해결책 (1 Pager) |
| `CoT_Note_질문북마크타임라인.docx` | 사고 과정 기록 (Abduction → AI → Gap → Takeaway) |

## 프로토타입 실행
- **가장 간단:** `prototype.html`을 더블클릭해 브라우저에서 연다.
- **로컬 서버:** 폴더에서 `python -m http.server 8137` 실행 후
  `http://localhost:8137/prototype.html` 접속.

## 테스트 순서 (before / after 체감)
1. 강의 재생(▶) — ‘A/B 테스트 기초: 통계적 유의성’ 더미 transcript.
2. **기존 채팅 방식**으로 전환 → 질문을 입력해 본다. 입력 중 자막이 계속 흘러
   ‘**놓친 자막 N줄**’이 집계된다(마찰 체감).
3. **질문 북마크 방식**으로 전환 → 막히는 순간 **‘막힘’ 버튼(또는 `Q`)** 탭.
   흐름은 멈추지 않고 타임라인에 맥락 카드가 핀으로 고정된다.
4. 카드를 열어 맥락 요약 + 질문 후보 3개 확인 → 선택하거나 직접 다듬어 전송.
   답변이 해당 타임스탬프 카드에 붙는다.
5. **복습 모드(⟲)** → 막힌 지점·해결 여부·Q&A가 시간순으로 정리된다.

단축키: `Q` 막힘 표시 · `K` 재생/정지.

## 상태 전이
`Listening → Marked → Question Candidate → Answered → Review`
